BATMAN Mesh Router — Architecture Overview

A Kotlin/JVM implementation of the BATMAN (Better Approach To Mobile Ad-hoc Networking) routing algorithm targeting heterogeneous, band-limited radio meshes. Each node discovers routes by flooding lightweight OGMs; no node ever computes full paths — only next-hop decisions are made locally.

2. OGM Flooding & Route Discovery

Every node periodically broadcasts an Originator Message (OGM) on each of its links. Relay nodes re-broadcast OGMs they haven't seen before, decrementing TTL and updating senderId to their own ID. The routing metric is implicit: a route's quality is the highest TTL remaining when the OGM arrives, which equals 15 − hop-count.

Relay suppression

  • Deduplication: SeenWindowCache uses a 64-bit sliding BitSet per originator, handling UShort (0–65535) wraparound via modular signed difference.
  • Jitter: relay is delayed by ogmInterval / (8 + rand(0,3)) so that copies from different paths arrive within the window and are counted.
  • Probabilistic suppression: after the jitter window, if N copies were heard, relay probability is 1/N — naturally suppresses redundant relays in dense meshes.
  • No relay-back: OGMs are never re-sent on the link they arrived on.

3. Multicast Groups & Spanning Tree

Multicast is tree-routed, not flooded. The tree is built implicitly by member beacons travelling toward the group owner — every relay node on the path records itself as a tree branch.

Stale branch eviction

Each LinkEntry in MulticastTree records lastSeen. A background loop calls evictStale(threshold) every threshold/3; branches whose member stopped sending beacons are pruned after at most threshold time.

4. Transmit Queue

A central pull-model queue decouples producers (routing logic) from consumers (per-link TX coroutines). Each link drains at its own pace; a slow radio never blocks a fast one.

Rule-based ordering

TxQueue accepts a List<Rule> (each a Comparator<TxEntry>) composed in order. The defaults are:

  1. controlFirst — OGM, BEACON, INVITE before MULTICAST, DATA
  2. olderFirst — FIFO within the same class
  3. insertionOrderFirst — strict total-order tiebreaker for TreeSet correctness

Eligible-link work stealing

Each TxEntry carries a Set<Link> of eligible radios rather than a single target. The first TX coroutine to call pollFor for any eligible link claims the frame exclusively. This enables natural load-balancing across redundant paths at zero extra cost.

5. Wire Format

All frames use a compact bit-packed layout. NodeIds are 12 bits (max 4095 nodes), TTL is 4 bits (max 15 hops), and the frame type occupies the upper nibble of byte 0. Authentication vs. encryption is a link-level property — no per-frame flag is needed.

FrameSizeSavings vs. naïve 16-bit layoutByte 0 upper nibble (type)

DataFrame7 + n B−3 B0x1

MulticastFrame9 + n B−3 B0x3
InviteFrame9 B−2 B0x4

Byte 0 layout shared by all frame types:

  bit 7–4 : type (4 bits, values 0–4 used, 5–15 reserved)
  bit 3–0 : firstId[11:8]  (high nibble of leading 12-bit identifier)

OGM layout example (6 bytes):

  B0: [type:4 | origId[11:8]:4]
  B1: origId[7:0]
  B2: [sendId[11:8]:4 | ttl[3:0]:4]
  B3: sendId[7:0]
  B4: seqNum[15:8]
  B5: seqNum[7:0]

GroupId is transmitted as its constituent fields (owner 12-bit + seq 16-bit) rather than as a raw 32-bit UInt, saving 4 bits. In memory it is a proper data class: data class GroupId(val owner: NodeId, val seq: UShort).

6. Key Design Decisions

DecisionRationale
OGM interval as routing metric Slower links emit OGMs less frequently → naturally accumulate lower TTL at destination → preferred only when no faster path exists. Zero extra overhead.
Pull-model TxQueue Each link's TX coroutine suspends when idle and wakes on demand via CompletableDeferred. Slow radios never block fast ones; no polling or busy-waiting.
1/N relay probability In dense subnets where N nodes all hear the same OGM, only 1/N relay on average — equivalent to a single relay in expectation, dramatically reducing OGM storms.
Beacon-based multicast tree Avoids flooding for multicast by having members build the tree themselves with periodic beacons. The owner only needs to forward along registered branches.
BitSet sliding window for dedup O(1) per lookup, bounded memory (64 bits per originator), handles UShort wraparound correctly via modular signed difference.
12-bit NodeId cap Mesh networks with band-limited radios rarely exceed a few hundred nodes. 12 bits (4095 nodes) is sufficient and saves 1 byte per NodeId field on the wire.
Neighbour table expiry: each entry expires after ogmInterval × neighborPurgeMultiplier (default 3×) without a refreshing OGM. This mirrors the batman-adv PURGE_TIMEOUT and ensures stale routes are evicted automatically after a node goes silent.
