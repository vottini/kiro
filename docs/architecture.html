BATMAN Mesh Router — Architecture Overview

A Kotlin/JVM implementation of the BATMAN (Better Approach To Mobile Ad-hoc Networking) routing algorithm targeting heterogeneous, band-limited radio meshes. Each node discovers routes by flooding lightweight OGMs; no node ever computes full paths — only next-hop decisions are made locally.

2. OGM Flooding & Route Discovery

Every node periodically broadcasts an Originator Message (OGM) on each of its links. Relay nodes re-broadcast OGMs they haven't seen before, decrementing TTL and updating senderId to their own ID. The routing metric is widest-path bandwidth: each OGM carries a minBandwidthTier: UByte field equal to floor(log₂(bps)) of the narrowest link traversed so far. The router selects the path with the highest bottleneck tier; TTL is used only as a tiebreaker.

Relay suppression

  • Deduplication: SeenWindowCache uses a 64-bit sliding BitSet per originator, handling UShort (0–65535) wraparound via modular signed difference.
  • Jitter: relay is delayed by ogmInterval / (8 + rand(0,3)) so that copies from different paths arrive within the window and are counted.
  • Probabilistic suppression: after the jitter window, if N copies were heard, relay probability is 1/N — naturally suppresses redundant relays in dense meshes.
  • No relay-back: OGMs are never re-sent on the link they arrived on.

3. Multicast Groups & Spanning Tree

Multicast is tree-routed, not flooded. The tree is built implicitly by member beacons travelling toward the group root — every relay node on the path records both legs of the beacon's journey in MulticastTree, progressively stitching together a spanning tree.

Tree construction and directional link tracking

MulticastTree tracks two independent sets of links per group:

  • Downstream — links from which beacons arrive (toward group members). Registered by registerDownstream when a beacon is received.
  • Upstream — the single link on which beacons are forwarded toward the root. Registered by registerUpstream, which replaces any previous upstream link rather than accumulating it. This ensures that after a route change, only the current best path is used — old upstream links do not linger and cause duplicate multicast transmissions.

When a MulticastFrame arrives on a link, linksFor(groupId, except = incomingLink) merges both sets and excludes the incoming link, so the frame flows in all valid directions without echo:

  • Arrives from a member below: forwarded upstream (toward root) and to sibling downstream branches.
  • Arrives from the root: forwarded to all downstream branches only.

Joining and leaving groups

Any node may join a multicast group directly by calling joinGroup(gid: GroupId, roots: List<NodeId>, beaconInterval: Duration). The roots list is an ordered list of candidate root nodes. If selfId is present in roots, the node acts as a root and does not start a beacon loop (it is already the convergence point). Otherwise, the node starts a beacon loop directed toward the first reachable root in the list. To stop participating, a node calls leaveGroup(gid: GroupId), which removes its membership and lets the beacon loop terminate naturally.

GroupId is a value class GroupId(val id: UInt) — an opaque 20-bit application-assigned value transmitted as 20 raw bits on the wire.

Leaf suppression

Only leaf nodes need to originate beacons. A relay node whose beacons from downstream members are already flowing through it does not need to send its own — the relayed beacons keep the upstream path alive and keep the relay's links registered in the tree.

At each tick, beaconLoop calls MulticastTree.hasActiveDownstream(groupId, 2 x beaconInterval). If any downstream link was refreshed within that window, the node skips its own beacon for that cycle. When all downstream members go silent and their entries are evicted, hasActiveDownstream returns false and the node resumes beaconing as a leaf, re-establishing its own branch.

Root failover

When the primary root is unreachable, the spanning tree has no convergence point. The roots: List<NodeId> passed to joinGroup serves as the ordered candidate list for failover — no separate invite mechanism or deputy list is needed.

  • Member side: beaconLoop calls resolveActiveRoot, which tries each entry in roots in order, returning the first one with a known route in neighborTable. The beacon is sent toward that candidate.
  • Relay side: relay nodes read frame.activeRoot to know where to forward the beacon. They need no local roots list — the active root is carried in the frame itself. Forwarding stops when frame.activeRoot == selfId.
  • Convergence: since all nodes share the same OGM-based reachability view, they independently agree on the same candidate, and the tree reconverges without coordination.

Stale branch eviction

Each LinkEntry in MulticastTree records lastSeen. A background loop calls evictStale(threshold) every threshold/3; branches whose member stopped sending beacons are pruned after at most threshold time. Both upstream and downstream entries are evicted independently.

4. Transmit Queue

A central pull-model queue decouples producers (routing logic) from consumers (per-link TX coroutines). Each link drains at its own pace; a slow radio never blocks a fast one.

Rule-based ordering

TxQueue accepts a List<Rule> (each a Comparator<TxEntry>) composed in order. The defaults are:

  1. controlFirst — OGM, BEACON before MULTICAST, DATA
  2. olderFirst — FIFO within the same class
  3. insertionOrderFirst — strict total-order tiebreaker for TreeSet correctness

Eligible-link work stealing

Each TxEntry carries a Set<Link> of eligible radios rather than a single target. The first TX coroutine to call pollFor for any eligible link claims the frame exclusively. This enables natural load-balancing across redundant paths at zero extra cost.

5. Wire Format

All frames use a compact bit-packed layout. NodeIds are 12 bits (max 4095 nodes), TTL is 4 bits (max 15 hops), and the frame type occupies the upper nibble of byte 0. Authentication vs. encryption is a link-level property — no per-frame flag is needed.

FrameSizeSavings vs. naïve 16-bit layoutByte 0 upper nibble (type)
OGM7 B−2 B0x0
DataFrame6 + varint + n B−1 B0x1
BeaconFrame8 B−2 B0x2
MulticastFrame7 + varint + n B−1 B0x3

n = payload bytes; payload length is encoded as a varint prefix. Type nibble values 0x4–0xF are reserved.

Byte 0 layout shared by all frame types:

  bit 7-4 : type (4 bits, values 0x0-0x3 used, 0x4-0xF reserved)
  bit 3-0 : firstId[11:8]  (high nibble of leading 12-bit identifier)

OGM layout (7 bytes):

  B0: [type:4 | origId[11:8]:4]
  B1: origId[7:0]
  B2: [sendId[11:8]:4 | ttl[3:0]:4]
  B3: sendId[7:0]
  B4: seqNum[15:8]
  B5: seqNum[7:0]
  B6: minBandwidthTier[7:0]

BeaconFrame layout (8 bytes), including activeRoot:

  B0: [type:4 | nextHop[11:8]:4]
  B1: nextHop[7:0]
  B2: [srcId[11:8]:4 | gid[19:16]:4]
  B3: srcId[7:0]
  B4: gid[15:8]
  B5: gid[7:0]
  B6: [activeRoot[11:8]:4 | spare:4]
  B7: activeRoot[7:0]

MulticastFrame layout (8 + n bytes):

  B0: [type:4 | srcId[11:8]:4]
  B1: srcId[7:0]
  B2: [gid[19:16]:4 | ttl[3:0]:4]
  B3: gid[15:8]
  B4: gid[7:0]
  B5: mcastSeq[15:8]
  B6: mcastSeq[7:0]
  B7...: varint(payloadLen) + payload

GroupId is a value class GroupId(val id: UInt) — an opaque 20-bit application-assigned value. It is transmitted as 20 raw bits spread across the beacon and multicast frame fields shown above (gid[19:16] in a nibble, gid[15:8] and gid[7:0] in the following two bytes).

6. Key Design Decisions

DecisionRationale
Widest-path bandwidth as routing metric (minBandwidthTier) Each OGM carries the bottleneck tier (floor(log₂(bps))) of the narrowest link traversed. Routers prefer the path with the highest tier; TTL breaks ties. This rewards high-throughput paths over low-latency-but-slow ones, which matters more for bulk transfers on band-limited radios.
"Best ever observed" tier per next-hop When OGMs from the same originator arrive via the same next-hop with varying tiers (relay suppression occasionally picks a slower intermediate hop), only lastSeen is refreshed; the stored tier is never lowered. The tier for a given next-hop can only increase until the entry expires. This trades accuracy for noise resistance: transient relay fluctuations don't degrade stored route quality, but a permanent upstream degradation won't be reflected until eviction and reinstall.
Pull-model TxQueue Each link's TX coroutine suspends when idle and wakes on demand via CompletableDeferred. Slow radios never block fast ones; no polling or busy-waiting.
1/N relay probability In dense subnets where N nodes all hear the same OGM, only 1/N relay on average — equivalent to a single relay in expectation, dramatically reducing OGM storms.
Beacon-based multicast tree Avoids flooding for multicast by having members build the tree themselves with periodic beacons. The root only needs to forward along registered branches.
Directional MulticastTree (upstream / downstream) Separating upstream (toward root) from downstream (toward members) links allows leaf suppression and prevents stale upstream links — a relay node can suppress its own beacon while still forwarding multicasts correctly in both directions.
Single upstream link (replace, not accumulate) registerUpstream replaces the current upstream link atomically. When the best route to root changes, the old link is discarded immediately rather than lingering until eviction — eliminating duplicate multicast transmissions during route transitions.
Leaf suppression in beaconLoop A relay node whose downstream members are actively beaconing through it does not need to originate its own beacon — the relayed beacons already keep the upstream path alive. This halves beacon traffic on every intermediate hop in deep trees.
Root failover via roots list in joinGroup No deputy list, no invite frame needed. The ordered roots list passed at join time doubles as the failover candidate list. resolveActiveRoot picks the first reachable entry; relay nodes carry the active root in each BeaconFrame so they need no local state. Since all nodes share the same OGM reachability view they converge on the same root independently.
BitSet sliding window for dedup O(1) per lookup, bounded memory (64 bits per originator), handles UShort wraparound correctly via modular signed difference.
12-bit NodeId cap Mesh networks with band-limited radios rarely exceed a few hundred nodes. 12 bits (4095 nodes) is sufficient and saves 1 byte per NodeId field on the wire.
Neighbour table expiry: each entry expires after ogmInterval × neighborPurgeMultiplier (default 3×) without a refreshing OGM. This mirrors the batman-adv PURGE_TIMEOUT and ensures stale routes are evicted automatically after a node goes silent.
